「グラフデータベース」の試作

RDB(リレーショナルデータベース)でもオブジェクト指向データベースでもXMLデータベースでもない
Altairの双方向グラフ構造データを基にした独特で個性的なデータベースの試作をしました。
これをGDB(グラフデータベース)と呼ぶことにします。これまでにないDML(データ操作言語)を備えています。
言語というと文字だけが並んだものというイメージがありますが、このDMLは記号文字を有効活用し、
問い合わせ文がグラフの図のように見えるというもので、ビジュアル的な効果もいくぶんあります。
このグラフデータベースは基本的には、
グラフデータ操作用「グラフ構造データ拡張Prolog言語処理系」です。

「グラフ構造データ拡張Prolog言語処理系」の修正

上記、「グラフデータベース」を実現するため、「グラフ構造データ拡張Prolog言語処理系」の
実装方式を少し修正しました。Prologには、述語と変数/定数がありますが、
これらをDAGにして実装していますが、述語を適用する場合、このDAGをコピーしていました。
単なるコピーだと、変数/定数を表すコピー前のノードとコピー後のノードは別物になりますが、
DAGのコピー前とコピー後で、変数/定数を表すノードを共有し、
コピー前のノードとコピー後のノードが同一ノードになるようにしました。

オセロゲーム（GbGpサンプル）の改良

Altairはインタプリタなので性能は悪いです。

GpGbサンプルのオセロゲームは、
形勢を先読みをする部分が頻繁に繰り返す所になっていて

Version 0.38 (2008/2/10)

グラフ構造データ拡張Prolog言語処理系

グラフ構造データのパターンパッチと置換によるProlog言語処理系を開発しました。
この処理系は、通常のPrologの実装より２〜１０倍遅いですが、
グラフ構造データを扱えるようにPrologを拡張するためにはやむお得ないです。
下記「汎用パーサー＆汎用ジェネレータ」で生成・作成するパースツリーは文法に依存しますが、
文法に依存しないグラフ構造データ（モデル）とパースツリー間の相互変換ができるように
Prologを拡張しました。Prologをベースにした理由は、出力データを入れると入力データが
同じプログラムで出てくる言語処理系がPrologだけだからです。
ある言語について、レガシーからエッセンスを抽出してモデルを構築するプログラムを作成すれば、
そのプログラムがそのまま、エッセンスだけを記述したモデルから既存言語のソースを生成する
ソフトウェア自動生成装置になる確率が増します。

汎用パーサー＆汎用ジェネレータ

文脈自由文法を記述する言語とその処理系を開発しました。
この言語で文法を記述するだけで、その文法に従ったテキストをパースするパーサー機能と、
その文法のパースツリーからその文法のテキストを生成するジェネレータ機能が実現できます。
下記の「Prologライクのリスト処理言語」のパースにこの汎用パーサを使用しています。
汎用ジェネレータ機能はソフトウェア自動生成に使用することが出来ます。

Prologライクのリスト処理言語の言語処理系

LISPの２進木のリスト処理の記述で、LISPのCAR/CDRを使用すると
プログラムの行数は増えるし、またプログラムの可読性が非常に悪いです。
LISPのCAR/CDRに相当する関数を使わなくても、Prologライクの記述で
リスト処理プログラムが記述できる言語とその処理系を開発しました。
LISPとの対比コードを参照してください。

Webシッピングサイトサンプル

簡易APサーバに、財務会計ソフトと連動した、
簡易Webショッピングシステムサンプルをつけました。

簡易RosettaNetサーバ(簡易APサーバ)

上記の機能を使った簡易RosettaNetサーバをつけました。
RosettaNetは、 企業間取引で使用するXMLの形式やプロトコルを
電子部品、情報機器の分野で標準化しようとしている団体が策定した仕様です。
今後、B2B分野で各パートナー企業のシステムを連携、統合した情報システムが
構築されていきますが、RosettaNetは、SOAPを使ったebXMLと並んで
そこで採用される標準の１候補となっています。

この簡易RosettaNetサーバはフリーソフトのLinuxとPostgresQLでも動作するので
SIコスト、保守などのサービスコストを少なく抑えることができれば
多額の投資ができない中小零細企業が少ない投資で使うことができます。
大規模システムでは構造がしばしば複雑になりすぎ、様々なコストが高くついたり
大幅な変更ができにくかったり、顧客の要求への対応に時間がかかったりしますが、
この簡易RosettaNetサーバは構造が比較的シンプルです。

ソフトの開発コストを高くする原因のひとつとして複雑になったシステムの環境設定があります。
環境設定が行いやすいように考えながら設定項目を厳選してソフトを開発していればよいのですが、
多くの場合は、開発速度やその他の優先事項を優先させるため、おもに開発者の都合で
設定項目の数が加速度的に増大し、環境設定作業が複雑になり、
結果として引き続き開発するSEの開発コストや高くなりがちで、
また高度な専門知識を持った人でないと、システムの運用できにくいという面があります。
この簡易RossetaNetサーバ(簡易APサーバ)はできるだけ環境設定が簡単になるようにしています。

OpenSSLを使った128bitのセキュアな通信と、OpenSSLのSMIMEを使った暗号化、署名もできます。

・サンプルGUIクラスライブラリのEdit/TextEditorからテキストを

Version 0.19 (2000/12/17)

RDBアクセス機能

ADBC(Altair DataBase Connection Interface)を策定し、
SQLによるRDBアクセス機能を付けました。
Oracle等のRDBアクセスを使ったアプリケーションが作成できます。
Windows版では、ODBC経由でアクセスします。
整数、実数、文字列に加えて日付やバイナリも扱えます。

Version 0.18 (2000/9/15)

Linux版の提供

Linux版を作成しました。
ただし、Windows版に比べ実装されていない機能が多々あります。
ほぼ動作するのは、GUI系基本部分と
Socketを使ったネットワーク系基本部分程度です。
また、日本語化はされていません。
動作させるにはGTK/GDK/X-Windowが必要です。

AlCtrlのWebページ

WebページにAltair OLEコントロールを貼りつけ、
WebサーバからAltairプログラムをロードし、
WebブラウザでAltairプログラムを動かせるようにしました。

Version 0.17 (2000/4/2)

暗号化機能

RSAベースの公開鍵暗号、および独自の共通鍵暗号の暗号化機能を付けました。
公開鍵暗号によって、共通鍵を暗号化して送ること、および、
認証を行うことができます。
公開鍵暗号で送信した共通鍵を使って平文を暗号化すること、および
暗号文を復号化することができます。

メッセージダイジェスト計算機能

MD5およびSHA-1のメッセージダイジェスト計算機能を付けました。
公開鍵暗号によって署名の確認を行う場合や
転送した内容が間違っていないかチェックするために使用します。

BigInteger演算機能

任意長の整数の四則演算ができる機能を付けました。

Version 0.16 (2000/2/11)

簡易HTTPサーバの追加

簡易HTTPサーバを付けました。
自分のPCをHTTPサーバにすることができます。
ActivePerlを入れることにより、CGIの簡易テストも行えます。

簡易STMPクライアントの追加

簡易STMPクライアントを付けました。
コマンドラインからテキストのメールを送付することが行えます。

Version 0.15 (1999/12/23)

DLL中の関数のネイティブ呼び出し機能の追加

基本的な型(char, short, long, float, double, ポインタ)を
in/out/inout引数および戻り値に持つDLL中の関数を呼び出す機能を
付けました。これにより、Windowsが持つOSの機能の呼び出しや、
DLLの形で提供されているコンポーネントの利用が容易になりました。

GUIクラスライブラリサンプルの改良

サンプルGUIクラスライブラリの階層構造エディタで、
ノードの折りたたみができるようにし、また、ビットマップを
ノード形状に使えるようにしました。

Version 0.14 (1999/11/3)

TCP/IP,UDPの通信を行うSocket関数の追加

TCP/IP, UDPプロコルで通信を行うことのできるSocket関数を追加しました。
これによりネットワークをまたいださまざまな
ネットワークアプリケーションを作成することが可能になります。

バイナリ型、バイナリでのファイル読み書きの導入

バイナリ型(任意の内容のバイト配列)と
バイナリ形式でのファイルの読み書きをサポートしました。
上記Socket関数でこれを用い、
任意の内容のファイル転送を行うことが可能になります。

GUIクラスライブラリサンプルのバグ修正

サンプルGUIクラスライブラリの階層構造エディタで、
並列処理で同時にデータを更新するとデータ内容が不正になるバグを
修正しました。

GUIリソースのキャッシュ化

Version 0.13以前では、描画に用いるGUIリソースの
ペン、ブラシ、フォントを毎回作成して破壊していました。
この方法だと、異なるペン、ブラシ、フォントで
多量の描画を行うとAPエラーを起こすことがありました。
(MFCのバグと思われます）
Version 0.14以降は、一度使ったペン、ブラシ、フォントをキャッシュ
するようにし、これによって問題が回避され、
また、描画自体も高速化されました。

Version 0.13 (1999/5/4)

XML形式ファイルパーサーとジェネレータ

XML形式のファイルをパーズしてパースツリーを作成するパーサー機能と
パースツリーの形式のツリーからXML形式のファイルを生成する
ジェネレータ機能をつけました。ジェネレータ機能はHTMLファイルの
自動生成に使うことが可能です。

プロセス間のリモートプロシージャコールサンプル

メモリマッピングオブジェクトとウィンドウメッセージを使って
プロセス間でリモートプロシージャコールを行うサンプルをつけました。
メモリマッピングオブジェクトとウィンドウメッセージの部分を
Socketを使ったものに書き換え、ネーミングサーバを用意すれば
CORBAやJava−RMIと同等のことができるようになります。

Java簡易IDEサンプル

Javaの簡易IDEをつけました。
プロジェクト管理、クラスブラウザ、ビルド、jdbをラップしたデバッガが使用できます。
プロジェクトファイルのファイルフォーマットはXML形式のファイルを使用しています。
ビルドはpackage分を抽出しパッケージに相当したディレクトリを自動的に掘ります。
デバッガでは、簡単なパーサによってクラス名を抽出しています。
JDKのソースにステップインすることが可能です。
テキストエディタは秀丸等のエディタを使用することができます。
GnuのRCSを使用してソースの版管理ができます。
Javaとの連携ではおもに標準入出力をパイプにつないで行っています。

SJIS/JIS/EUCコード変換

SJISとJIS、SJISとEUCの間で文字コードを互いに変換するコード変換機能を付けました。

Version 0.12 (1998/11/23)

レジストリ操作OLEコントロール

Windowsのレジストリを操作するOLEコントロールをつけました。

コマンド引数、プロセス終了コード、パイプ、メモリマップトファイル

Altairにコマンド引数を与えてAltairAPの中からそれを参照できるようにしました。
AltairをAP中から終了させる場合、プロセス終了コードを設定できるようにしました。
Altairから別のEXEを起動させる場合に、そのEXEの標準入出力をAltairAPから
読み書きできるようにするパイプを提供しました。
プロセス間でメモリマップトファイル(共有メモリ)を介してデータを送受信できるように
しました。

ウィンドウメッセージ

Windowsのウィンドウメッセージ通信機能をつけました。
異なるプロセス間でメッセージのやり取りを行うことができます。

Altairスクリプト

クラスに変更を加えることなしに動的に関数をロードし実行させるための機能を設けました。
通常の関数はクラスのメンバーになっていなければなりませんが、
これだと、関数をロードするたびにクラスに動的に変更を加えなければならず
不便なことがあります。

OLEサーバー化

AltairをOLEサーバー化しました。OLEオートメーションによって
Altairの機能の一部をAltair外部から使用できるようにしました。

C/C++のソースからexeをビルドするサンプル

C/C++のソースからexeをビルドするサンプルをつけました。
Altairでソース生成を行いC/C++のアプリケーションを開発する手がかりになります。

Version 0.10 (1997/12/23)

GpGbサンプルのマニュアル

GbGpサンプル(GUIビルダー/結線プログラミング)の簡易マニュアルをつけました。

OLEオートメーションの改善

引数付きプロパティセットを可能にしました。

Version 0.08 (1997/2/11)

マルチメディアOLEコントロールの提供

Altairに依存しない汎用のマルチメディアOLEコントロールを提供しました。
このOLEコントロールを使うことによってAVIファイル、WAVEファイル、

AltairをOCXコンテナとした簡単なサンプルをつけています。

Altair OLEコントロールの提供

Altairで作成したプログラムをOLE Control(OCX)で実行できるOCX(AlCtrl)を

VB等のOCXコンテナで実行することができるようになります。

Version 0.05 (1996/9/30)

MDI/SDIフレームウィンドウ

複数のMDIフレームウィンドウとSDIフレームウィンドウを
出すことができるようになりました。
ツールバーは出せませんがメニューバーは出すことができます。

GUIビルダーの部品名のメンバー変数化(GbGpサンプル)

GbGpサンプルのGUIビルダーの部品名をGUIビルダーの派生クラスの
メンバー変数にしました。これにより、GbGpサンプルのプログラムが
行ないやすくなりました。

サブクラス化した部品の配置(GbGpサンプル)

GbGpサンプルのGUIビルダー部分で、ユーザがカスタマイズした部品を
メニューに登録し配置できるようになりました。

階層化ノードのコピーペースト(GbGpサンプル)

GbGpサンプルの結線プログラム画面で、選択したノード/アークの
コピー/ペーストができるようになりました。ノードが階層化ノードでも
ノードをコピー/ペーストすることができます。

Version 0.04 (1996/8/15)

OCXコンテナ化

AltairをOCXコンテナにするための機能を追加しました。
OCXを埋め込むことができ、OLEオートメーションによって
コンテナ(Altair)側から埋め込んだOCXの機能をそのまま使えるように
なります。ただし、基盤部のみなので実際に使用するためにはサンプルに
あるようなプログラミングが必要です。

Version 0.03 (1996/5/3)

Altairプログラムの生成インターフェース

サンプルGbGpでは、２次元のデーターフロー的な図からAltairソースを
生成していますが、0.00ではこれをVC++で記述していました。
0.03では、プログラムを生成するインターフェースを公開し、
AltairのプログラムでAltairのソースを生成することができるようになりました。
これによりリフレクションが可能になります。

GUI画面の生成もファイルからのシリアライズ方式から
生成ソースの実行に変更しました。

テキスト２形式ライブラリ

派生関係にない複数のクラスを含むライブラリを作成することができる
ようになりました。

al_exec関数の導入

インタプリタの中からWindowsのAPを起動できるようになりました。
