Altairのデバッグ機能

第0.43版 2016年10月23日
第0.00版 1996年 2月11日

概要

Altairは、インタプリタ実行でエラーが発生した場合、
そのエラーがどこで発生したかの特定ができるようになっています。
Altairにはブレークポイントやトレースの機能はありませんが、
プログラム実行中にプリント文等を適宜挿入することにより
デバッグ情報を得ることができます。
  • エラーを起こした場所の特定
  • 実行中のプリント文挿入

エラーを起こした場所の特定

ある場所でインタプリタエラーが発生するとアウトプットウィンドウに
以下の情報が出力されます。
  • スタックトレース
  • エラーを起こした行から10数行のソース
  • 簡単なメッセージ
たとえば、以下のようなソースコードを挿入すると必ずエラーになりますが
var list ls;
ls.head = 1;
この場合アウトプットウィンドウには以下のような出力がなされます。
        Rectangle::Draw
        MonoGlyph::Draw
        TMGlyph::Draw
        PolyGlyph::Draw
        GlyphEditor::Draw
        MonoGlyph::Draw
        RootGlyph::Draw
=========================
  ls.head = 1;
  al_push_pen(w, line_style, line_width, line_color);
  al_push_brush(w, fill_style, fill_color, fill_pattern);
  al_rectangle(w, left, top, right, bottom);
  al_pop_brush(w);
  al_pop_pen(w);
}
END
インタプリタ実行でエラーが起きました。(SetHead)
注: インタプリタ実行中に上記のフォーマット以外の
エラーメッセージが出た場合はAltairをできるだけすみやかに
終了してください。
Altairはその他のエラー処理があまいので落ちる可能性があります。

実行中のプリント文挿入

Altairではよぼどのことがない限り、プログラム実行中にソースを
修正することができます。
ブレークポイントで止めて変数の値を参照する代わりに、
適宜プリント文を挿入することによりそれの
代わりをさせることがほとんどすべての場合に可能です。
